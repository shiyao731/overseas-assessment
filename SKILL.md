## 🎓 理论框架（v3.0.0 新增）

你的评估基于以下权威理论框架：

### 1. OLI 折衷范式（Eclectic Paradigm）

**提出者**：John H. Dunning (1977)

跨国经营是三类优势的组合：
- **O (Ownership)** - 所有权优势：企业级竞争力与资源
- **L (Location)** - 区位优势：目标国市场机会与制度环境
- **I (Internalization)** - 内部化优势：进入模式与治理结构

**应用**：
- O 不足 → 企业准备度不够
- L 不利 → 目标国环境不友好
- I 不匹配 → 进入方式不合理

### 2. CAGE 距离框架

**提出者**：Pankaj Ghemawat (2001, HBR)

国际化面临"距离"带来的摩擦与成本：
- **C (Cultural)** - 文化距离：宗教、语言、社会规范
- **A (Administrative)** - 行政距离：制度、政策、殖民历史
- **G (Geographic)** - 地理距离：物理距离、时差、气候
- **E (Economic)** - 经济距离：收入水平、成本差异

**应用**：用于量化文化维度和评估国际化难度

### 3. Uppsala 国际化过程模型

**提出者**：Johanson & Vahlne (1977)

国际化是"知识积累—承诺增加"的渐进过程：
- 阶段1：偶尔出口
- 阶段2：通过代理商出口
- 阶段3：建立销售子公司
- 阶段4：生产设施本地化

**应用**：输出分阶段可行路径，而非仅给出总分

## ⚠️ Gating 机制（v3.0.0 新增 - 重要！）

**核心升级**：将"不可进入"（No-Go）从加权平均中剥离出来

### Gating 检查流程（必须在评分之前执行！）

```python
def gating_check(f, c, m):
    """
    f: 企业
    c: 目标国
    m: 进入模式

    返回: {
        "passed": True/False,
        "blockers": [阻断原因列表],
        "warnings": [警告列表],
        "score_adjustments": {维度: 扣分}
    }
    """

    # 步骤1：外资持股/准入禁令（No-Go）
    if not check_foreign_ownership(f, c, m):
        return {"passed": False, "blockers": ["外资准入限制"], ...}

    # 步骤2：产品级市场准入与 NTMs（No-Go）
    if not check_product_access(f, c, m):
        return {"passed": False, "blockers": ["产品准入限制"], ...}

    # 步骤3：关键牌照/数据合规（No-Go）
    if not check_licensing_data(f, c, m):
        return {"passed": False, "blockers": ["关键牌照缺失"], ...}

    # 步骤4：其他硬门槛（Warning/降级，不触发 No-Go）
    warnings, adjustments = check_other_constraints(f, c, m)

    return {"passed": True, "warnings": warnings, "score_adjustments": adjustments}
```

**重要**：Gating 检查必须在评分之前执行，如果 Gating 失败，直接输出 No-Go，不计算总分！

## 🎲 不确定性量化（v3.0.0 新增）

**升级**：从"点估计"到"区间估计 + 覆盖度"

### Monte Carlo 模拟

**原理**：
1. 将每个指标视为分布（而非点值）
2. 重复抽样 10000 次
3. 输出分位数区间 [P10, P90]

**示例**：
```
v2.1.3: 总分 62.3（点估计）
v3.0.0: 总分 62.3，区间 [54.2, 70.1]（P10-P90）
```

### 覆盖度评分

**定义**：衡量"有多少重要信息真实可得"

```python
C = (Σ W_d · 𝕜(该维度关键指标可用)) / (Σ W_d)
```

### 评级标准（v3.0.0 新标准）

| 条件（全部满足） | 评级 |
|---------|------|
| **S ≥ 75 且 S_{P10} ≥ 65 且 C ≥ 0.75 且无 blockers** | 🟢 **Go**（出海条件成熟） |
| **55 ≤ S < 75 或区间过宽或覆盖度不足** | 🟡 **Caution**（谨慎进入） |
| **S < 55 或存在 blockers** | 🔴 **No-Go**（暂不具备条件） |

### 行动清单生成逻辑

```python
def generate_action_plan(company_profile, evaluation_result, competitive_intel):
    """
    基于企业画像、评估结果、竞品情报生成行动清单
    """
    actions = []

    # 类别1：联系人建议
    if company_profile["stage"] == "初次出海":
        actions.append({
            "timeframe": "0-1个月",
            "category": "联系人",
            "action": f"建议联系 {company_profile['name']} 的国际业务部/战略部负责人",
            "talking_points": ["贵司出海规划进展", "目标市场初步调研", "合规准备情况"],
            "priority": "high"
        })

    # 类别2：展会/活动建议
    target_market = evaluation_result["recommended_market"]
    actions.append({
        "timeframe": "2-4个月",
        "category": "展会活动",
        "action": f"推荐参加 {target_market} 相关行业展会",
        "specific_events": search_trade_fairs(target_market, company_profile["industry"]),
        "priority": "medium"
    })

    # 类别3：知识服务提供
    high_risk_items = [d for d in evaluation_result["dimensions"] if d["risk_level"] == "high"]
    if high_risk_items:
        for item in high_risk_items:
            actions.append({
                "timeframe": "0-2个月",
                "category": "知识服务",
                "action": f"提供关于 {item['name']} 的合规简报",
                "content_brief": item["risk_description"],
                "priority": "high"
            })

    # 类别4：竞品对标话术
    if competitive_intel["recent_moves"]:
        for intel in competitive_intel["recent_moves"]:
            actions.append({
                "timeframe": "立即",
                "category": "竞品对标",
                "action": f"提及竞争对手 {intel['competitor']} 在 {intel['location']} 的最新动态",
                "talking_points": [
                    f"您的竞争对手 {intel['competitor']} 上个月在 {intel['location']} 设立了{intel['entity_type']}",
                    "我们可以帮助您快速跟进或采取差异化策略"
                ],
                "priority": "high"
            })

    return sorted(actions, key=lambda x: x["priority"])
```

**核心升级**：双层级（子公司+母公司）竞品情报收集

### 竞品情报收集流程

```
第一步：识别竞争对手
├── 同行业直接竞对
├── 跨行业潜在竞对（相似产品/服务）
└── 目标市场本地竞对

第二步：双层级情报扫描
├── 竞对子公司层面
│   ├── 搜索: "XX公司 + 海外/国际/出口/投资"
│   ├── 收集: 设厂时间、地点、规模、业务类型
│   └── 来源: 新闻公告、年报、招股书
└── 竞对母公司层面
    ├── 搜索: "XX集团 + 出海/国际化战略"
    ├── 收集: 集团整体出海布局、投资动态
    └── 来源: 集团官网、投资者关系、行业报告

第三步：情报整合与分析
├── 时间线梳理: 按时间顺序排列竞对出海事件
├── 地图可视化: 标注竞对在全球的布局
└── 对标点识别: 找出可利用的竞争动态
```

### 竞品情报输出格式

```markdown

### 直接竞争对手出海动态

|---------|-------|------|------|------|------|------|

### 集团层面战略对比
- **XX集团**: 积极推进欧洲本地化生产，已建立3个海外基地
```

## 👔 关键决策人推断系统（v3.1.0 新增）

**核心升级**：基于企业画像推荐应联系部门和关键决策人

### 决策人推断逻辑

```python
def recommend_contacts(company_profile, evaluation_result):
    """
    基于企业画像和评估结果推荐接触点
    """
    contacts = []

    # 规则1：根据出海阶段推荐部门
    stage_dept_map = {
        "初次出海": ["国际业务部", "战略部", "董秘办"],
        "已有海外项目": ["海外运营部", "采购部", "人力资源部"],
        "大型集团": ["战略投资部", "董事会办公室", "国际业务单元"],
        "中小型创新企业": ["CEO办公室", "战略部", "市场部"]
    }

    for dept in stage_dept_map.get(company_profile["stage"], []):
        # 搜索企业是否有该部门
        dept_info = search_company_department(company_profile["name"], dept)
        if dept_info:
            contacts.append({
                "department": dept,
                "contact_role": infer_role(dept, company_profile["stage"]),
                "talking_points": generate_topics(dept, evaluation_result),
                "urgency": "high" if dept in ["国际业务部", "战略部"] else "medium"
            })

    return contacts
```

### 部门-角色-话题映射表

| 出海阶段 | 推荐部门 | 关键角色 | 沟通话题 | 紧急度 |
|---------|---------|---------|---------|-------|
| 初次出海 | 国际业务部 | 国际业务总监 | 目标市场选择、合规准备、时间规划 | 🔴 高 |
| 初次出海 | 战略部 | 战略经理 | 出海战略定位、竞品分析、投资回报 | 🔴 高 |

| 大型集团 | 战略投资部 | 投资总监 | 集团出海布局、投资机会评估 | 🔴 高 |
| 中小创新 | CEO办公室 | CEO/创始人 | 快速增长策略、资源匹配 | 🔴 高 |
