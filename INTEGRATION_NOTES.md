# overseas-assessment 与 web-access 整合说明

## 整合概述

本次更新将 **web-access skill** 的核心能力整合到 **overseas-assessment** 中，大幅提升了企业出海评估的智能检索能力和效率。

---

## 核心能力提升

### 1. 三层智能检索架构

整合前：仅使用 WebSearch 工具
整合后：智能选择最佳检索工具

```
┌─────────────────────────────────────────────────────────────┐
│                    三层检索架构                               │
├─────────────────────────────────────────────────────────────┤
│  第一层：WebSearch (Duckduckgo/Bing/Brave)                   │
│  • 用途：一般性搜索、发现信息源                               │
│  • 场景：企业概况、行业数据、新闻动态                         │
├─────────────────────────────────────────────────────────────┤
│  第二层：Jina Reader (jina.ai/http://)                       │
│  • 用途：已知 URL 的高效内容提取                              │
│  • 场景：官网页面、研报文章、政策文件                         │
│  • 优势：Token 效率高、处理动态页面                           │
├─────────────────────────────────────────────────────────────┤
│  第三层：浏览器自动化 (CDP/Playwright)                        │
│  • 用途：处理反爬虫网站、需要登录的内容                       │
│  • 场景：企业征信平台、招聘网站、政府门户                     │
│  • 优势：突破访问限制、保持登录状态                           │
└─────────────────────────────────────────────────────────────┘
```

### 2. 目标导向研究流程

整合前：固定四轮搜索步骤
整合后：以研究目标为核心的动态流程

```
研究目标体系
├── 目标1：构建企业完整画像
├── 目标2：评估出海基础能力
├── 目标3：分析目标市场环境
└── 目标4：识别风险与机遇
```

### 3. 站点经验系统

新增 **site-patterns** 目录，包含常用中文商业平台的访问指南：

| 文件 | 内容 |
|------|------|
| `tianyancha.md` | 天眼查访问指南 |
| `qcc.md` | 企查查访问指南 |
| `company-sites.md` | 公司官网通用访问指南 |
| `job-platforms.md` | 招聘平台访问指南 |
| `news-sources.md` | 新闻媒体检索指南 |

### 4. Token 效率优化

通过 Jina Reader 提取网页内容：
- **传统方式**: 直接读取网页可能消耗 5000+ tokens
- **Jina 方式**: 通常 < 2000 tokens
- **节省比例**: 约 60%

### 5. 并行研究能力

对于复杂企业评估，可使用 Task 工具启动多个子代理并行研究：
- 子代理 A：企业财务与融资
- 子代理 B：行业与市场竞争
- 子代理 C：出海动态与政策
- 子代理 D：目标市场环境

## 使用示例对比

### 整合前
```
用户：评估比亚迪的出海可行性

执行流程：
1. WebSearch: "比亚迪 海外业务"
2. WebSearch: "比亚迪 行业数据"
3. WebSearch: "比亚迪 出海动态"
4. WebSearch: "目标市场政策"
→ 固定步骤，无法根据发现调整策略
```

### 整合后
```
用户：评估比亚迪的出海可行性

执行流程（目标导向）：
1. 目标1：构建企业画像
   - WebSearch: "比亚迪 官网 财务"
   - Jina Reader: https://jina.ai/http://www.byd.com/
   - Jina Reader: https://jina.ai/http://www.tianyancha.com/ + "比亚迪"

2. 目标2：评估出海能力
   - WebSearch: "比亚迪 海外业务 出口"
   - WebSearch: "比亚迪 技术专利"

3. 目标3：分析市场环境
   - WebSearch: "欧洲 新能源汽车 政策"

4. 目标4：识别风险
   - WebSearch: "欧盟 反补贴调查"

→ 灵活调整，智能选择工具
```

## 技术要点

### Jina Reader 使用方法
```bash
# 基本格式
https://jina.ai/http://{原始URL}

# 示例
https://jina.ai/http://www.byd.com
https://jina.ai/http://www.tianyancha.com/search?key=比亚迪
```

### 浏览器自动化场景
- 天眼查/企查查（反爬虫）
- 猎聘/拉勾（可能需要登录）
- 政府网站（访问限制）
- 复杂单页应用
